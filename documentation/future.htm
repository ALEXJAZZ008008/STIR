Future developments

The STIR library, in its current state, possesses many capabilities. The developers, however, look forward to still further increases in the flexibility and power of the software. Some of the developments being discussed are:
  • expanded library of polymorphic classes (e.g. image grids, and ordered subsets)
  • additional scanners and modalities: HiDAC, DHCI, SPECT
  • additional data formats support, without conversion to Interfile.
  • listmode reconstruction algorithms
Below, we list of some of the more concrete features planned for the next releases. However, which feature is actually finalised/implemented depends on the needs of the developers. If you want one of these features and are willing to help, let us know.
  • More automatic testing programmes
  • More algorithms: OSCB (potentially ART, OSMD), FORE coming soon
  • More projectors
  • More priors
  • Parallelisation (using PVM or EPX)
  • Compatibility of the interpolating backprojector with SPARC and Opteron processors.
  • Allow images of arbitrary size in z-direction as well.
  • Allow x,y offset in images (at least for some projectors)
  • Automatic segmentation

Note that some of these involve updating existing PARAPET programs which were written in a (private and incompatible) earlier release.

Finally, as far as coding concerns, the following (big) changes are planned (but see todo sections in the documentation or the files)

  • Change Reconstruction hierarchy to use objective functions. Also template them in terms of the input data format (necessary for listmode reconstructions)
